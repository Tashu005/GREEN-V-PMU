Clock Constraints (XDC)
create_clock -period 2.0 -name clk_in [get_ports clk_in] for 500 MHz master. Use MMCM to generate from 100 MHz board oscillator. Route clk_out through BUFGCE for glitch-free output.

I/O Port Mapping
ibuf_occupancy[6:0] → 7 DIP switches or GPIO. inst_window → BRAM fed from soft-CPU. volt_ctrl[2:0] → 3 GPIO → PMIC (TPS65xxx I²C). clk_out → FPGA clock output pin to MCU CLKIN.

Probe: scale_cmd scale_valid spike_detected freq_sel change_req change_ack. Trigger condition: spike_detected==1 to capture lookahead event. Compare pre- and post-spike freq_sel to prove 0-cycle prediction.

Synthesize → Implement → Bitstream
Synthesis: expect <200 LUTs, <50 FFs. Timing report: all paths must close at 500 MHz. Check for latches (should be zero — design is fully registered). Program via JTAG. Verify current_sel matches freq_sel after change_ack.
MCU HARDWARE-IN-THE-LOOP DEMO
1
Clock Chain: FPGA → MCU
FPGA clk_out → MCU SYSCLK input pin. MCU runs zero clock management firmware — Green-V controls its clock entirely. Connect volt_ctrl[2:0] → 3-bit R-2R DAC → MCU VDD regulator enable pins.
2
Instruction Feed via SWD/SPI
Tap MCU instruction fetch bus via SWD debug port → SPI bridge → FPGA inst_window[7:0][31:0]. Only need lower 7 bits (opcode field) per instruction. Alternatively: pre-load test patterns in FPGA BRAM for repeatable demo.
3
Power Measurement (INA226)
Insert INA226 on MCU VDD rail (I²C current sensor, 1mΩ shunt). Sample at 1 kHz via second MCU/Raspberry Pi. Compare: (A) fixed max freq, (B) Linux cpufreq SW governor, (C) Green-V. Expect 40-62% reduction on bursty workload.
4
Oscilloscope Demo Script
Trigger on spike_detected rising edge. Channel 1: inst_window[0][6:0] (opcode bus). Channel 2: freq_sel[2:0]. Show freq_sel rising BEFORE occupancy crosses threshold. This is the visual proof of 0ns predictive reaction time.
